"상관관계(히트맵)"을 선택하면 전체 변수의 상과계수가 표시된 히트맵을 그려줘

Python Flask를 활용해서 **예측 시스템** 웹 애플리케이션 전체 코드를 생성해줘.
웹 시스템은 전체적으로 **한글 UI**로 구현해줘.

# 2. 개발 개요

파이썬 Flask를 활용해서 파일 업로드부터 변수 선택, 결측치 제거, 이상치 탐색, 탐색적 분석, 회귀분석, 모델 검증, 시뮬레이션까지 순차적으로 진행되는 예측 시스템을 만들어줘.

이 시스템은 전 단계의 결과가 다음 단계에서 사용되는 구조여야 한다.

예를 들어:

* 파일 업로드 결과는 변수 선택 단계에서 사용
* 변수 선택 결과는 결측치 제거 단계에서 사용
* 결측치 제거 결과는 이상치 탐색 단계에서 사용
* 이상치 제거 결과는 탐색적 분석 및 회귀분석에서 사용
* 회귀분석 결과는 모델 검증 및 시뮬레이션에서 사용

세션이 유지될 수 있도록 전 단계 결과를 SQLite3 데이터베이스에 저장하고, 다음 단계에서 SQLite3에 저장된 결과를 불러와 사용하게 해줘.

그래프를 그릴 때 한글이 깨지지 않도록 matplotlib 한글 폰트 설정을 반드시 포함해줘.
운영체제별로 가능한 한글 폰트를 자동으로 설정하도록 작성해줘.

# 4. 화면 구성

전체 화면은 좌측 메뉴 영역과 우측 작업 영역으로 구성해줘.

## 4-1. 화면 비율

화면을 2분할해줘.

* 좌측 화면: 전체 가로폭의 20%
* 우측 화면: 전체 가로폭의 80%

우측 화면은 다시 상단과 하단으로 분할해줘.

* 우측 상단: 우측 화면 높이의 10%
* 우측 하단: 우측 화면 높이의 90%

## 4-2. UI 컬러

사용자 인터페이스의 주요 컬러는 다음 색상을 중심으로 만들어줘.

* 연한 녹색
* 연한 파란색
* 회색

디자인은 가독성 좋고 초보자도 쉽게 사용할 수 있게 구성해줘.

## 4-3. 좌측 메뉴 구성

좌측 화면에는 다음 메뉴를 세로로 표시해줘.

1. 파일 업로드
2. 변수 선택
3. 결측치 제거
4. 이상치 탐색
5. 탐색적 분석
6. 회귀분석
7. 모델 검증
8. 시뮬레이션

## 4-4. 메뉴 진행 방식

좌측 메뉴는 반드시 순서대로 진행되어야 한다.

초기 상태에서는 다음과 같이 표시해줘.

* "1. 파일 업로드" 메뉴만 진하게 표시
* 나머지 메뉴는 연하게 표시
* 아직 실행할 수 없는 메뉴는 비활성화 상태처럼 보이게 표시

각 메뉴가 성공적으로 실행되면 다음 메뉴가 진하게 표시되어 클릭 가능하도록 해줘.

예시:

* 파일 업로드 완료 후 → "2. 변수 선택" 활성화
* 변수 선택 완료 후 → "3. 결측치 제거" 활성화
* 결측치 제거 완료 후 → "4. 이상치 탐색" 활성화
* 이상치 탐색 완료 후 → "5. 탐색적 분석" 활성화
* 탐색적 분석 완료 또는 건너뛰기 가능 처리 후 → "6. 회귀분석" 활성화
* 회귀분석 완료 후 → "7. 모델 검증" 활성화
* 모델 검증 완료 후 → "8. 시뮬레이션" 활성화

현재 단계는 메뉴에서 강조 표시해줘.

## 4-5. 우측 화면 구성

좌측 메뉴를 클릭했을 때 우측 화면은 다음처럼 구성해줘.

### 우측 상단 영역

* 현재 단계명을 표시
* 예시: `2. 변수 선택`
* 해당 단계에서 필요한 선택 메뉴, 버튼, 설정값 등을 표시

### 우측 하단 영역

* 서브메뉴
* 실행 결과
* 표
* 그래프
* 안내 메시지

# 5. 데이터 저장 및 세션 유지

이 시스템은 단계별 결과가 다음 단계로 이어져야 하므로 SQLite3 데이터베이스에 데이터를 저장해야 한다.

## 5-1. SQLite3 저장 원칙

다음 정보를 SQLite3에 저장해줘.

* 업로드된 원본 데이터
* 현재 분석용 데이터
* 선택된 종속변수
* 선택된 독립변수
* 결측치 제거 결과
* 이상치 탐색 결과
* 이상치 제거 결과
* 탐색적 분석 실행 이력
* 회귀분석 결과
* 사용자가 선택한 최종 모델
* 모델 검증 결과
* 시뮬레이션 입력값 및 예측값

데이터프레임은 SQLite 테이블에 저장하거나 pickle 파일 경로를 SQLite에 저장하는 방식 중 안정적인 방식을 선택해서 구현해줘.

대용량 데이터를 고려해서 가능하면 다음 방식을 추천한다.

* 실제 데이터프레임은 `data/session_data.pkl` 파일로 저장
* SQLite3에는 파일 경로, 단계 상태, 선택 변수, 분석 결과 요약, 모델 결과 등을 저장

## 5-2. 세션 관리

Flask session 또는 고유 session_id를 사용해서 사용자의 분석 흐름을 구분해줘.

각 사용자의 분석 상태를 다음처럼 관리해줘.

* session_id
* current_step
* upload_completed
* variable_selected
* missing_processed
* outlier_processed
* eda_completed
* regression_completed
* validation_completed
* simulation_completed

# 6. 메뉴별 상세 기능

---

## 6-1. 1. 파일 업로드

"1. 파일 업로드" 메뉴를 클릭하면 사용자가 컴퓨터 폴더에서 CSV 또는 Excel 파일을 업로드할 수 있게 해줘.

### 업로드 가능 파일

* `.csv`
* `.xlsx`
* `.xls`

### 기능 요구사항

* CSV 파일은 `pandas.read_csv()`로 읽어줘.
* Excel 파일은 `pandas.read_excel()`로 읽어줘.
* 업로드된 파일은 서버의 `uploads/` 폴더에 저장해줘.
* 업로드된 데이터를 pandas DataFrame으로 읽어와 `data/session_data.pkl` 또는 세션별 pkl 파일로 저장해줘.
* SQLite3에는 업로드 파일명, 저장 경로, 데이터 행 수, 열 수, 컬럼명, 현재 단계 상태를 저장해줘.
* 파일 업로드가 완료되면 다음 메뉴인 "2. 변수 선택"이 활성화되도록 해줘.

### 출력 요구사항

우측 화면 하단에 다음 내용을 출력해줘.

* 업로드 파일명
* 전체 행 수
* 전체 열 수
* 컬럼 목록
* pandas `head(10)`을 사용한 상위 10개 행
* pandas `describe(include="all")`을 사용한 변수별 기초 통계량

### 예외 처리

다음 상황을 처리해줘.

* 파일을 선택하지 않은 경우
* 지원하지 않는 파일 형식인 경우
* 파일을 읽을 수 없는 경우
* 빈 데이터 파일인 경우

---

## 6-2. 2. 변수 선택

"2. 변수 선택" 메뉴를 클릭하면 우측 화면 상단에 종속변수와 독립변수를 선택하는 메뉴를 만들어줘.

### 기능 요구사항

* 업로드된 데이터의 컬럼명을 자동으로 불러와 선택 목록을 만들어줘.
* 종속변수는 하나만 선택할 수 있게 해줘.
* 독립변수는 여러 개 선택할 수 있게 해줘.
* 선택된 종속변수와 독립변수는 SQLite3에 저장해줘.
* 이후 모든 분석은 사용자가 선택한 변수 기준으로 진행해줘.
* 변수 선택 완료 후 "3. 결측치 제거" 메뉴가 활성화되도록 해줘.

### 출력 요구사항

우측 화면 하단에 다음 내용을 출력해줘.

* 선택한 종속변수
* 선택한 독립변수 목록
* 분석에 사용될 전체 변수 목록
* 분석 대상 데이터의 행 수와 열 수

### 예외 처리

다음 상황을 처리해줘.

* 종속변수를 선택하지 않은 경우
* 독립변수를 하나도 선택하지 않은 경우
* 종속변수가 독립변수에 중복 선택된 경우
* 선택한 변수가 데이터에 존재하지 않는 경우

---

## 6-3. 3. 결측치 제거

"3. 결측치 제거" 메뉴를 클릭하면 선택된 분석 변수 기준으로 결측치가 있는 행을 찾고 제거해줘.

### 기능 요구사항

* 분석 대상 컬럼은 선택된 종속변수와 독립변수만 사용해줘.
* 선택된 변수 중 하나라도 결측치가 있으면 해당 행을 결측치 행으로 판단해줘.
* 결측치가 있는 행 수를 계산해줘.
* 결측치가 있는 행을 제거해줘.
* 결측치 제거 후 현재 분석용 데이터프레임을 업데이트하고 저장해줘.
* SQLite3에 결측치 제거 전 행 수, 제거된 행 수, 제거 후 행 수를 저장해줘.
* 결측치 제거 완료 후 "4. 이상치 탐색" 메뉴가 활성화되도록 해줘.

### 출력 요구사항

우측 화면 하단에 다음 내용을 출력해줘.

* 결측치 제거 전 전체 행 수
* 결측치가 포함된 행 수
* 제거된 행 수
* 결측치 제거 후 남은 행 수
* 결측치 제거 후 데이터 미리보기 상위 10개 행

예시 출력:

* 기존 데이터 행 수: 1,000개
* 결측치가 있는 행 수: 35개
* 제거된 행 수: 35개
* 현재 남은 행 수: 965개

### 예외 처리

다음 상황을 처리해줘.

* 파일 업로드가 되지 않은 경우
* 변수 선택이 되지 않은 경우
* 결측치 제거 후 데이터가 비어버리는 경우

## 6-4. 4. 이상치 탐색

"4. 이상치 탐색" 메뉴를 클릭하면 각 변수별 이상치를 탐색해줘.

### 이상치 기준

각 숫자형 변수별로 다음 범위 밖에 존재하는 데이터를 이상치로 판단해줘.

* 하한값 = 평균 - 3 × 표준편차
* 상한값 = 평균 + 3 × 표준편차

즉, 다음 조건에 해당하면 이상치이다.

* 값 < 평균 - 3 × 표준편차
* 값 > 평균 + 3 × 표준편차

### 기능 요구사항

* 이상치 탐색은 선택된 종속변수와 독립변수 중 숫자형 변수만 대상으로 해줘.
* 문자열 변수나 범주형 변수는 이상치 탐색 대상에서 제외해줘.
* 표준편차가 0이거나 계산할 수 없는 변수는 제외해줘.
* 변수별 평균, 표준편차, 하한값, 상한값, 이상치 개수를 계산해줘.
* 이상치 탐색 결과는 SQLite3에 저장해줘.
* 우측 화면 상단에 변수별 이상치 제거 여부를 선택할 수 있는 체크박스 또는 선택 메뉴를 표시해줘.
* 사용자가 특정 변수의 이상치 제거를 선택하면 해당 변수에서 이상치가 있는 행을 제거해줘.
* 여러 변수를 선택한 경우 선택된 변수 중 하나라도 이상치가 있는 행은 제거해줘.
* 이상치 제거 후 현재 분석용 데이터프레임을 업데이트하고 저장해줘.
* 이상치 탐색 또는 제거 완료 후 "5. 탐색적 분석" 메뉴가 활성화되도록 해줘.

### 출력 요구사항

우측 화면 하단에 다음 내용을 출력해줘.

* 변수별 이상치 탐색 결과 표

  * 변수명
  * 평균
  * 표준편차
  * 하한값
  * 상한값
  * 이상치 개수
* 이상치 제거 전 전체 행 수
* 제거된 이상치 행 수
* 이상치 제거 후 남은 행 수
* 이상치 제거 후 데이터 미리보기 상위 10개 행

### 예외 처리

다음 상황을 처리해줘.

* 숫자형 변수가 없는 경우
* 이상치 제거 후 데이터가 비어버리는 경우
* 파일 업로드 또는 변수 선택이 완료되지 않은 경우

* 릿지
* 라쏘

* 사용자가 선택한 종속변수와 독립변수를 기준으로 회귀분석을 수행해줘.

  * `random_state=42`

## 6-8. 8. 시뮬레이션

"8. 시뮬레이션" 메뉴를 클릭하면 만들어진 최종 회귀모형을 사용자가 직접 시뮬레이션할 수 있도록 해줘.

### 기능 요구사항

* SQLite3에서 사용자가 선택한 최종 회귀모형을 불러와줘.
* 우측 화면 상단에는 사용자가 선택한 회귀식을 출력해줘.
* 우측 화면 하단에는 독립변수를 슬라이더바로 입력할 수 있게 해줘.
* 각 독립변수마다 슬라이더바를 하나씩 만들어줘.
* 슬라이더의 최소값과 최대값은 현재 분석용 데이터에서 해당 변수의 최소값, 최대값을 기준으로 자동 설정해줘.
* 슬라이더 기본값은 해당 변수의 평균값으로 설정해줘.
* 슬라이더 값이 변경되면 종속변수 예측값이 자동으로 변경되게 해줘.
* 예측 결과는 숫자와 막대그래프로 동시에 출력해줘.
* 한 화면에서 슬라이더 입력, 종속변수 예측값, 막대그래프가 모두 보이도록 가독성 있게 구성해줘.
* 인터랙티브한 그래프 출력을 위해 Plotly와 Dash 패키지를 사용해줘.
* Flask 앱 안에서 Dash 앱을 연동하는 방식으로 구현해줘.
* 시뮬레이션 입력값과 예측값은 SQLite3에 저장해줘.

### 출력 요구사항

우측 상단:

* 최종 선택 모델명
* 최종 회귀식
* 종속변수명

우측 하단:

* 독립변수별 슬라이더바
* 현재 입력값 표
* 예측된 종속변수 값
* Plotly 막대그래프

### 예외 처리

다음 상황을 처리해줘.

* 최종 모델이 선택되지 않은 경우
* 모델 파일을 불러올 수 없는 경우
* 슬라이더를 만들 수 없는 독립변수가 있는 경우
* 범주형 변수가 포함되어 시뮬레이션 입력이 어려운 경우에는 해당 범주형 변수는 드롭다운으로 선택할 수 있게 해줘.

# 7. 추가 기능 요구사항

아래 기능도 필요하면 함께 추가해줘.

## 7-1. 단계 진행 상태 저장

* 현재 단계 완료 여부를 SQLite3에 저장해줘.
* 사용자가 새로고침해도 이전 단계 결과가 유지되게 해줘.
* 완료된 메뉴는 체크 표시를 보여줘.
* 현재 진행 가능한 메뉴는 진하게 표시해줘.
* 아직 진행할 수 없는 메뉴는 비활성화 표시해줘.

## 7-2. 초기화 기능

* 전체 분석을 처음부터 다시 시작할 수 있는 "초기화" 버튼을 만들어줘.
* 초기화 버튼을 누르면 다음 내용을 삭제해줘.

  * SQLite3에 저장된 세션 상태
  * 업로드된 파일
  * 임시 데이터 파일
  * 생성된 그래프 파일
  * 저장된 모델 파일

## 7-3. 분석 로그 기능

* 사용자가 각 단계에서 실행한 작업 로그를 SQLite3에 저장해줘.
* 예:

  * 파일 업로드 완료
  * 변수 선택 완료
  * 결측치 제거 완료
  * 이상치 제거 완료
  * 회귀분석 실행
  * 최종 모델 선택
  * 모델 검증 완료
  * 시뮬레이션 실행

## 7-4. 결과 다운로드 기능

* 분석 결과를 CSV 또는 Excel로 다운로드할 수 있게 해줘.
* 회귀분석 결과와 모델 검증 결과를 다운로드할 수 있게 해줘.
* 현재 분석용 데이터를 다운로드할 수 있게 해줘.

# 12. 구현 세부사항

## 12-1. 한글 폰트 설정

matplotlib 그래프에서 한글이 깨지지 않도록 다음 로직을 포함해줘.

* Windows: Malgun Gothic
* macOS: AppleGothic
* Linux: NanumGothic 또는 DejaVu Sans fallback
* 마이너스 기호가 깨지지 않도록 `plt.rcParams["axes.unicode_minus"] = False` 설정

## 12-2. 표 출력

pandas DataFrame은 다음처럼 HTML 테이블로 변환해서 보여줘.

* `df.head(10).to_html(classes="table table-striped table-hover table-sm", index=False)`
* `df.describe(include="all").to_html(classes="table table-bordered table-sm")`

## 12-3. 그래프 저장

그래프는 다음 중 하나의 방식으로 구현해줘.

* `static/plots/` 폴더에 PNG 파일로 저장
* 또는 base64 문자열로 변환해서 HTML에서 출력

가능하면 파일명 충돌 방지를 위해 session_id와 UUID를 조합해서 파일명을 만들어줘.

## 12-4. 회귀분석 데이터 전처리

* 종속변수는 숫자형이어야 한다.
* 독립변수는 숫자형과 범주형을 모두 허용한다.
* 범주형 독립변수는 `pd.get_dummies(drop_first=True)`로 변환한다.
* train/test split을 적용한다.
* StandardScaler는 릿지와 라쏘에만 적용한다.
* 전진선택법과 후진제거법은 statsmodels OLS 기반으로 구현한다.
* 결측치 및 이상치 처리 후 데이터가 너무 적으면 경고를 표시한다.

## 12-5. 모델 저장

최종 선택 모델은 다음 정보를 함께 저장해줘.

* 모델 객체
* 전처리 컬럼 목록
* 더미 변수 컬럼 목록
* 스케일러 객체
* 종속변수명
* 독립변수명
* 회귀식 문자열
* 성능 지표

joblib 또는 pickle을 사용해서 `data/models/` 폴더에 저장해줘.
