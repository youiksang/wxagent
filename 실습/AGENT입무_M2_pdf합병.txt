## PDF 분할/병합 프로그램

---

# 2. 개발 목표

사용자가 PDF 파일을 업로드한 뒤 페이지를 미리보기로 확인하고, 원하는 방식으로 PDF를 분할하거나 여러 PDF 파일을 병합할 수 있는 데스크톱 앱을 만든다.

---

# 3. 기술 스택

다음 패키지를 사용해줘.

* Python
* PyQt5
* pypdf 또는 PyPDF2
* PyMuPDF 또는 pdf2image

PDF 처리에는 `pypdf` 패키지를 우선 사용해줘.
PDF 썸네일 미리보기에는 구현이 쉬운 방식으로 `PyMuPDF(fitz)`를 사용해줘.

---

# 4. 주요 기능

## 4-1. PDF 업로드 기능

* 사용자가 버튼을 클릭하면 파일 선택 창이 열리게 해줘.
* PDF 파일만 선택할 수 있게 해줘.
* 선택된 PDF 파일 경로를 화면에 표시해줘.
* PDF 업로드 후 전체 페이지 수를 표시해줘.
* 업로드된 PDF의 페이지 썸네일을 화면에 미리보기로 표시해줘.
* 썸네일은 스크롤 영역에 표시해줘.
* 각 썸네일 아래에는 페이지 번호를 표시해줘.

---

## 4-2. PDF 미리보기 기능

PDF를 업로드하면 페이지별 썸네일 미리보기를 보여줘.

요구사항:

* PDF 각 페이지를 이미지로 변환해서 썸네일로 표시해줘.
* 미리보기 영역은 스크롤 가능해야 해.
* 페이지 수가 많아도 화면이 멈추지 않도록 가능하면 썸네일 크기를 적절히 줄여줘.
* 썸네일 크기는 가로 150px 정도로 설정해줘.
* 페이지 번호는 `1페이지`, `2페이지` 형식으로 표시해줘.

---

# 5. PDF 분할 기능

분할 방식은 다음 3가지를 지원해줘.

## 5-1. 범위 입력 분할

사용자가 페이지 범위를 입력하면 해당 범위만 새 PDF로 저장해줘.

예시 입력:

```text
1-3
```

위 입력은 1페이지부터 3페이지까지 추출해서 하나의 PDF 파일로 저장한다.

요구사항:

* 입력값은 `시작페이지-끝페이지` 형식을 사용해줘.
* 페이지 번호는 사용자 기준으로 1부터 시작하게 해줘.
* 실제 PDF 내부 처리는 0부터 시작하는 인덱스로 변환해줘.
* 시작 페이지가 끝 페이지보다 크면 오류 메시지를 보여줘.
* 전체 페이지 수를 초과하면 오류 메시지를 보여줘.
* 저장 위치는 사용자가 직접 선택할 수 있게 해줘.

---

## 5-2. 페이지 입력 분할

사용자가 특정 페이지 번호들을 입력하면 해당 페이지만 새 PDF로 저장해줘.

예시 입력:

```text
1,3,5
```

위 입력은 1페이지, 3페이지, 5페이지만 추출해서 하나의 PDF 파일로 저장한다.

요구사항:

* 쉼표로 구분된 페이지 번호를 입력받아 처리해줘.
* 공백이 있어도 처리되게 해줘.
* 중복 페이지 번호가 있으면 한 번만 처리해줘.
* 페이지 번호가 전체 페이지 수를 초과하면 오류 메시지를 보여줘.
* 저장 위치는 사용자가 직접 선택할 수 있게 해줘.

---

## 5-3. 고정 페이지 단위 분할

사용자가 숫자를 입력하면 해당 페이지 수 단위로 PDF를 여러 개로 나눠 저장해줘.

예시 입력:

```text
2
```

전체 7페이지 PDF라면 다음처럼 저장된다.

* 1_2.pdf
* 3_4.pdf
* 5_6.pdf
* 7_7.pdf

요구사항:

* 사용자가 입력한 숫자만큼 페이지를 묶어서 여러 PDF로 분할해줘.
* 예: 2를 입력하면 2페이지씩 분할
* 저장 폴더는 사용자가 선택할 수 있게 해줘.
* 저장 파일명은 원본파일명_시작페이지_끝페이지.pdf 형식으로 만들어줘.
* 입력값이 1보다 작으면 오류 메시지를 보여줘.

---

# 6. PDF 병합 기능

사용자가 여러 PDF 파일을 선택하고 하나의 PDF로 병합할 수 있게 해줘.

## 6-1. 파일 선택

* 병합할 PDF 파일을 여러 개 선택할 수 있게 해줘.
* 선택된 파일 목록을 화면에 표시해줘.
* 파일 목록에는 파일명과 전체 경로를 표시해줘.
* 선택 순서대로 병합되도록 해줘.

## 6-2. 파일 순서 조정

가능하면 선택된 PDF 파일의 병합 순서를 조정할 수 있게 해줘.

버튼을 추가해줘.

* 위로 이동
* 아래로 이동
* 선택 파일 제거
* 전체 목록 초기화

## 6-3. 병합 실행

* 사용자가 "병합하기" 버튼을 누르면 선택한 순서대로 PDF를 병합해줘.
* 저장 위치와 파일명은 사용자가 직접 선택하게 해줘.
* 병합이 완료되면 완료 메시지를 보여줘.

---

# 7. 화면 구성

추천 화면 구성:

* 상단: 앱 제목
* 왼쪽 영역: PDF 업로드 및 미리보기

## 7-1. 메인 레이아웃

## 7-2. 왼쪽 영역

다음 요소를 포함해줘.

* PDF 업로드 버튼
* 선택된 PDF 경로 표시 라벨
* 전체 페이지 수 표시 라벨

포함 요소:

* 아래로 이동 버튼

# 8. 예외 처리

다음 예외 처리를 반드시 포함해줘.

* PDF 파일을 업로드하지 않고 분할 버튼을 누른 경우
* PDF 파일이 아닌 파일을 선택한 경우
* PDF 파일을 열 수 없는 경우
* 페이지 범위 입력 형식이 잘못된 경우
* 페이지 번호 입력 형식이 잘못된 경우
* 페이지 번호가 1보다 작은 경우
* 페이지 번호가 전체 페이지 수보다 큰 경우
* 저장 위치를 선택하지 않은 경우
* 병합할 PDF 파일을 선택하지 않은 경우
* 병합할 PDF 파일이 2개 미만인 경우
* 병합 중 오류가 발생한 경우
* 암호화된 PDF나 읽기 어려운 PDF를 처리할 수 없는 경우에는 안내 메시지를 보여줘.

# 11. 구현 세부사항

## 11-1. PDF 처리

* 우선 `pypdf`의 `PdfReader`, `PdfWriter`, `PdfMerger`를 사용해줘.
* pypdf import가 실패하면 `PyPDF2`를 사용하도록 fallback 코드를 작성해줘.

## 11-2. 썸네일 생성

* PyMuPDF를 사용해서 각 페이지를 이미지로 렌더링해줘.
* 각 페이지의 pixmap을 QImage로 변환한 뒤 QPixmap으로 표시해줘.
* 썸네일은 너무 크지 않게 비율을 유지해서 축소해줘.
* 페이지가 많은 PDF도 고려해서 썸네일 생성 중 오류가 나면 해당 페이지는 건너뛰고 안내 메시지를 남겨줘.

## 11-3. 저장 파일명

* 범위 분할 저장 파일은 사용자가 직접 지정
* 페이지 입력 분할 저장 파일은 사용자가 직접 지정
* 고정 페이지 단위 분할 저장 파일은 자동 생성
* 자동 생성 형식:

  * 원본파일명_시작페이지_끝페이지.pdf
